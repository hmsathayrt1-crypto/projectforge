## Critical Issues 🔴

### 1) Static assets not served at expected paths
**Severity:** Critical  
**Evidence:**
- `GET /styles.css` → 404
- `GET /app.js` → 404
- Files are available at `/public/styles.css` and `/public/app.js`

**Impact:** Main UI styling and JS logic fail on root path deployment; app UX/functionality is broken for normal load path.

**Likely root cause:** Deployment done from `.` with `public/` nested static files, while HTML references `styles.css` and `app.js` as root-relative/relative assets.

---

### 2) Missing `/api/auth/me` endpoint
**Severity:** Critical  
**Evidence:** `GET /api/auth/me` → 404

**Impact:** Session restore/profile fetch fails (`fetchCurrentUser()` and profile page logic depend on this route).

---

### 3) Missing `/api/teams/match` endpoint
**Severity:** High  
**Evidence:** `GET /api/teams/match` → 404

**Impact:** Team matching feature does not use real backend; frontend falls back to mock behavior.

---

## High / Medium Issues 🟠

### 4) Route contract mismatch in auth file design
`functions/api/auth/register.ts` contains GET logic that behaves like `me`, but route path is `/api/auth/register` (not `/api/auth/me`).

**Impact:** Confusing API contract and maintenance risk.

---

### 5) `OPTIONS` not handled on APIs
**Evidence:** `OPTIONS /api/projects` → 405

**Impact:** Preflight/CORS interoperability issues for non-simple cross-origin clients.

---

### 6) Duplicate app initialization
`public/app.js` calls `initApp()` twice:
- once on `DOMContentLoaded`
- once directly at file end

**Impact:** duplicate listeners, repeated requests, flaky UI behavior.

---

### 7) Non-existent import paths in auth functions
Files import:
- `@cloudflare/workers-workers/sha256` (not present)

**Impact:** build fragility / dead code smell; future build/runtime risk.

---

### 8) API fallback-to-mock behavior masks backend failures
Frontend silently falls back to mock data for several failures.

**Impact:** QA can look “green” while backend integration is actually broken.

---

## Security / Hardening Notes 🟡
- Password hashing uses plain SHA-256 without salt/slow KDF (should use PBKDF2/Argon2/bcrypt-like strategy in Workers context).
- No rate limiting observed on auth endpoints.
- Token lifecycle/invalidation is basic (acceptable for MVP but needs hardening for production).
